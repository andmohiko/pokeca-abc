# Phase 1: 初回提出（ルールベースエージェント）

## 目標

カードプールからデッキを1つ組み、ルールベースのエージェントを実装して初回提出を完了する。

## 作業ステップ

### Step 1: デッキを組む

- カードプール（約2,000種）から60枚のデッキを構築する
- デッキコンセプト（メインアタッカー、サブアタッカー、サポート構成）を決める
- `deck.csv` にカードID×60行で記述する

### Step 2: デッキの強い動きを言語化する

- 理想的なゲーム展開（先攻/後攻）を言語化する
- 各ターンで何を優先すべきかを整理する
  - 序盤: 盤面展開（ベンチにポケモンを並べる、進化準備）
  - 中盤: 攻撃準備（エネルギー加速、進化完了）
  - 終盤: サイド取り切り（ボスの指令でターゲット選択）
- カードごとの役割と使いどころを明確にする
- 特定の相手・状況への対応方針を決める

### Step 3: デッキの動きをスコア化して実装する

- Step 2で言語化した戦略をスコアリングロジックに変換する
- サンプルコードの設計パターンに従い実装する
  - 各OptionTypeに対するスコア付け
  - ターン内の行動優先度（進化 > アイテム > サポート > エネルギー > 攻撃）
  - 攻撃プラン（誰で誰を殴るか）の最適化
- `main.py` にエージェントを実装する
- Kaggle Notebookにコードを貼り付けて提出する

## 完了条件

- [x] デッキコンセプトが決まっている
- [x] デッキの動きが言語化されている
- [x] `deck.csv` が作成されている
- [x] `main.py` にルールベースエージェントが実装されている
- [x] Kaggleに初回提出が完了している
