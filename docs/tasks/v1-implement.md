# V1 Step3: スコア化と実装 — ソウブレイズexデッキ

## 実装方針

サンプルコード（Dragapult ex / Mega Lucario ex）のアーキテクチャを踏襲し、ソウブレイズexデッキの戦略をスコアリングロジックに変換した。

### 基本アーキテクチャ

```
agent(obs_dict)
  → Observationに変換
  → select.option の各選択肢にスコアを付ける
  → スコア降順で maxCount 個を返す
```

## スコア設計

### ターン内の行動優先度（スコア帯）

スコアは「行動の強さ」ではなく「ターン内での実行順序の優先度」を表す。

| スコア帯 | 行動 | 理由 |
|---|---|---|
| 100,000 | ディスカード時のエネルギー選択 | しんえんほむらの火力に直結。最優先でエネルギーをトラッシュに送る |
| 80,000 | アンフェアスタンプ（きぜつ直後）、相手スタジアム上書き | 妨害は早めに実行 |
| 70,000 | 進化（カルボウ→ソウブレイズex） | 進化を先にすると後続の行動を制御しやすい |
| 50,000〜53,000 | ポケモンをベンチに出す | 盤面構築は早めに |
| 46,000〜48,000 | なかよしポフィン、ハイパーボール | 展開・サーチ系グッズ |
| 40,000〜42,000 | クラッシュハンマー、夜のタンカ、特性使用 | グッズ・特性 |
| 35,000〜38,000 | サポート使用（ゼイユ/アオキ/ボス等）、いれかえ、スタジアム配置 | サポートは1ターン1枚制約 |
| 25,000〜30,000 | エネルギー手貼り（ソウブレイズexに優先） | 攻撃準備 |
| 14,000 | リーリエの決心 | 他サポートがない場合のフォールバック |
| 10,000 | にげる | 必要な場合のみ |
| 1,000〜1,200 | 攻撃（しんえんほむら/アメジストレイジ） | ターン最後の行動 |
| -1 | 不要な行動 | スキップ可能なら選択しない |
| -10,000,000 | 絶対に不要 | 完全に無駄な行動 |

### ソウブレイズex固有のスコアリング

#### ディスカード（コスト支払い時）

ハイパーボールのコストやゼイユの効果でカードをトラッシュする際、**エネルギーを最優先でトラッシュ**する（スコア100,000）。これがソウブレイズexデッキの核心で、サンプルコードとの最大の差異。

```
エネルギー → スコア100,000（最優先でトラッシュ）
不要カード → hand_scoreの逆数
重要カード → トラッシュしない（負のスコア）
```

#### エネルギー手貼り

一般的なデッキと異なり、**エネルギーは手札に温存してトラッシュ手段で捨てる方が強い**。手貼りは以下の条件のみ：

- ソウブレイズexにエネルギーが0枚 → 付ける（攻撃に必要な炎1個）
- ソウブレイズexにエネルギーが1枚以上 → 付けない（トラッシュに回す）
- カルボウがアクティブでベンチに攻撃可能なポケモンがいない → 付ける（ちからをあつめる用）

#### 攻撃選択

ソウブレイズexには2つのワザがある：

| ワザ | コスト | ダメージ | 効果 |
|---|---|---|---|
| しんえんほむら | 炎 | 30 + トラッシュエネ×20 | エネルギー消費なし |
| アメジストレイジ | 炎超鋼 | 280 | 付いているエネルギーを全トラッシュ |

基本はしんえんほむらを使用。アメジストレイジは3色エネルギーが必要なため、現在のデッキ（炎単色）ではほぼ使用しない。

#### サポート選択

1ターンに1枚のみ使えるため、事前に最適なサポートを決定する：

| 優先度 | サポート | 条件 |
|---|---|---|
| 1 | ボスの指令 | 相手ベンチに倒したいexがいる |
| 2 | ゼイユ | 手札にエネルギーがある（トラッシュ+ドロー） |
| 3 | アオキの手際 | 手札にエネルギーが多い（全トラッシュ+サーチ） |
| 4 | リーリエの決心 | 上記が使えない場合のドローソース |

### 状況別のスコア調整

#### 先攻 vs 後攻

- **先攻選択**（IS_FIRSTでYESにスコア1）
- **後攻の場合**: SETUP_ACTIVE_POKEMONでスボミーに最高スコア（100,000）を付与し、グッズロックを狙う

#### きぜつ直後（pre_ko = True）

- アンフェアスタンプ: スコア80,000（最優先で使用）
- キチキギスex: 特性「さかてにとる」で自動ドロー
- 夜のタンカ: ソウブレイズexの回収スコアが上昇

## 主要関数

| 関数 | 役割 |
|---|---|
| `agent()` | メインエントリーポイント。obs_dictを受け取り、行動インデックスリストを返す |
| `_hand_score()` | 手札の各カードの価値を評価（サーチ先選択・ディスカード判断に使用） |
| `_attach_score()` | エネルギー/道具の付け先を評価 |
| `pokemon_score()` | 相手ポケモンのターゲット価値を評価（サイド枚数・エネルギー・進化段階を考慮） |
| `prize_count()` | きぜつ時のサイド取得枚数を計算（Legacy Energy等の補正込み） |
| `calc_damage()` | トラッシュのエネルギー枚数からしんえんほむらのダメージを算出 |
| `set_card_counts()` / `add_card_count()` | デッキ残り枚数を追跡（サーチ判断に使用） |
| `get_card()` | 盤面の指定エリアからカード/ポケモンオブジェクトを取得 |

## テスト・提出方法

### Kaggle Notebookでの手順

1. シミュレーション部門のNotebookを作成
2. Inputに `pokemon-tcg-ai-battle` のCompetition Dataを追加
3. セル1: `deck.csv` をPythonリストから作成
4. セル2: `%%writefile main.py` でエージェントコードを書き出し
5. セル3: `submission.tar.gz` を作成（main.py + deck.csv + cg/）
6. 「Save & Run All」→ Outputの `submission.tar.gz` を「Submit to Competition」
